# Security Policy

## Supported Versions
- Only the latest release is supported for security updates.

## Reporting a Vulnerability
- Please report security issues via GitHub Security Advisories or email the maintainers directly.
- Do not disclose vulnerabilities publicly until a fix is released.

## Best Practices
- Follow [OWASP Top Ten](https://owasp.org/www-project-top-ten/) for secure coding.
- Use environment variables and AWS Secrets Manager for all credentials.
- Run Trivy and Checkov scans before merging to main.

## Response
- We aim to respond to all security reports within 48 hours.
